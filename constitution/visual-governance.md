# Visual Governance Rules (Tree-Brain Model)

## 1. Multi-Process, Multi-Agent Contextual Zoom

The system must maintain a dynamic, real-time, infinitely zoomable representation of all processes, sub-processes, agents, and dependencies. A single node may participate in multiple larger processes simultaneously, and the visualization must reflect all of these relationships.

This includes:
- multi-tasking
- multi-agent collaboration
- cross-departmental workflows
- shared resources and responsibilities
- overlapping timelines
- hierarchical and non-hierarchical structures

The visualization must always show how a node fits into the whole organism, how it contributes to multiple branches, and how it affects the system’s direction.

## 2. Natural Language Node Translation

The system must translate any node the user is viewing into a simple, beautiful, human-readable description in a few sentences. This description must capture:

- the node’s purpose
- its parent processes
- its child processes
- its relationships
- its dependencies
- its current state
- its future trajectory
- its significance within the organism

This rule ensures that both humans and AI agents share the same mental model of the system’s structure and evolution.
